## 4. Write a `README.md`

Cover these sections, prose-style:

**What it is.** A personal Congress bill tracking dashboard. Pulls bills from the Congress.gov API, summarizes them with Gemini, displays them in a Bloomberg-Terminal-style feed.

**Live demo.** Link to `https://cbt-chi-silk.vercel.app`.

**Stack.** Next.js 15 (App Router), TypeScript, Tailwind, Turso (libSQL), Gemini API, deployed to Vercel.

**Running locally.** Walk through:
1. `npm install`
2. `cp .env.example .env` and fill in keys (with notes on where to get each)
3. `npm run migrate` to create tables
4. `npm run sync` to fetch bills
5. `npm run summarize` to generate summaries (note: takes ~20 minutes for full backfill, ~$1 in Gemini API cost)
6. `npm run dev` to start the dashboard

**Deployment.** Deployed via Vercel with a daily cron at 09:00 UTC that runs sync + summarize for new bills.

**Project structure.** Quick directory overview: `app/` (Next.js pages), `lib/` (shared logic), `scripts/` (standalone CLI scripts), `.claude/skills/cbt/` (LLM context for working on the project), `docs/handoffs/` (development notes).

**Acknowledgments.** Mention the data comes from Congress.gov's public API, summaries are generated by Gemini.

Keep it under 100 lines, no marketing language.
