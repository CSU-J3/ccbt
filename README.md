# CCBT — Colorado Capitol Bill Terminal

A personal Colorado state bill tracking dashboard. Pulls bills from the
[Plural Open](https://open.pluralpolicy.com) (OpenStates v3) API, summarizes
them with Gemini, and displays them in a Bloomberg-Terminal-style feed with
topic and stage filters plus a watchlist.

Forked from [CBT](https://github.com/CSU-J3/cbt) (Congress Bill Terminal),
which tracks federal bills.

## Stack

Next.js 15 (App Router) · TypeScript · Tailwind v4 · Turso (libSQL) ·
`@google/genai` (Gemini 2.5 Flash) · Vercel hosting + Cron.

## Running locally

1. `npm install`
2. `cp .env.example .env` and fill in:
   - `OPENSTATES_API_KEY` — sign up at https://open.pluralpolicy.com (free
     tier is 500 req/day, 10/sec; bronze tier at 5,000/day available by
     emailing `contact@openstates.org`)
   - `GEMINI_API_KEY` — generate at https://aistudio.google.com/apikey
   - `TURSO_DATABASE_URL` and `TURSO_AUTH_TOKEN` — create a database with the
     [Turso CLI](https://docs.turso.tech/cli/installation) (`turso db create ccbt`,
     then `turso db tokens create ccbt`)
   - `CRON_SECRET` — any 32-byte hex string,
     e.g. `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`
   - `CO_CURRENT_SESSION` — the active Colorado legislative session
     identifier (e.g. `2025A`). Update this when sessions roll over; verify
     against `GET /jurisdictions/ocd-jurisdiction/country:us/state:co/government?include=legislative_sessions`.
3. `npm run migrate` to create the `bills` and `watchlist` tables
4. `npm run sync` to fetch Colorado bills from OpenStates (~700 bills per
   regular session; incremental on subsequent runs)
5. `npm run summarize` to generate plain-English summaries with Gemini
   *(deferred — replaced in a later handoff with state-bill-aware prompts)*
6. `npm run dev` — dashboard at http://localhost:3000

## Deployment

Deployed via `vercel --prod`. A daily Vercel Cron at 09:00 UTC (`vercel.json`)
hits `/api/sync` with a `Bearer ${CRON_SECRET}` header; the route runs sync
first, then summarizes any new NULL-summary rows (capped at 50/tick to fit the
Hobby 60-second function ceiling).

## Project structure

- `app/` — Next.js App Router pages (`/`, `/bill/[id]`, `/watchlist`) and API
  routes (`/api/sync`, `/api/watchlist`)
- `components/` — small Tailwind primitives (`BillRow`, `StageIndicator`,
  `TopicTags`, `HeaderBar`, `FooterLegend`, …)
- `lib/` — shared logic: `db.ts`, `queries.ts`, `sync.ts`, `stage.ts`,
  `summarize.ts`, `summarize-runner.ts`, `enums.ts`, `topic-colors.ts`,
  `format.ts`
- `scripts/` — standalone CLI entry points: `migrate.ts`, `sync.ts`,
  `summarize.ts`
- `.claude/skills/cbt/` — `SKILL.md` with project context for Claude Code
  (still labeled `cbt` — to be re-skinned for CCBT in a later handoff)
- `docs/handoffs/` — chronological handoff prompts that drove each step of
  development, kept for posterity
